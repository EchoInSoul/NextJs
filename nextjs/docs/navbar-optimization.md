# Navbar 优化与规范文档

本文档记录了对 `Navbar` 组件及其配置进行的性能优化、结构重构和路由规范化工作。

## 1. 性能优化

针对用户反馈的网页卡顿问题，我们对 `Navbar.tsx` 进行了深度优化，主要解决了 JS 主线程被高频事件阻塞的问题。

### 1.1 移除高频状态更新
- **移除 `scrollPercent`**: 之前在每次 `scroll` 事件中都会计算页面滚动百分比并触发 `setScrollPercent`，导致组件每秒重渲染数十次。该功能已被移除，因为“回到顶部”按钮已不再显示百分比数字。
- **优化 `isScrolled` 检测**: 保留了防抖 (debounce) 处理，仅在滚动超过 50px 时切换状态，大幅减少了状态更新频率。

### 1.2 简化交互逻辑
- **回到顶部按钮**: 移除了复杂的百分比计算逻辑，现在仅作为纯粹的功能按钮：
  - 点击时平滑滚动回顶部（若在首页）。
  - 跳转回首页（若不在首页）。

## 2. 组件结构重构

### 2.1 左侧分组菜单 (Group Menu)
- **扁平化展示**: 移除了“我的”、“推荐”等分组标题，将所有子菜单项合并展示。
- **Grid 布局**: 采用 `grid-cols-2` 双列布局，使菜单更加紧凑整齐。
- **视觉优化**: 
  - 增加了菜单项的内边距 (`py-2`)，提升点击区域高度。
  - 移除了分组间的分割线 (`border-b`)，界面更清爽。

### 2.2 中间主菜单 (Main Menu)
- **单列布局**: 下拉菜单恢复为单列 Flex 列表，符合传统下拉菜单习惯。
- **样式精简**: 移除了菜单项右侧的下箭头图标 (`fa-chevron-down`)。
- **圆角统一**: 将主菜单按钮圆角调整为 **8px**，与 ZEROPOINT 主题风格保持一致（下拉容器保持 12px）。

### 2.3 首页入口
- **隐藏文字**: 左上角的 Site Name 隐藏了文字，仅保留 Home 图标。
- **智能导航**: 点击图标时，如果在首页则平滑回顶，否则跳转至首页。

## 3. 路由规范化

### 3.1 路径标准化
- 修改了 `config/nav.config.ts`，移除了所有路径末尾的斜杠（Trailing Slash）。
- **变更示例**:
  - `/about/` -> `/about`
  - `/archives/` -> `/archives`
- **目的**: 符合 Next.js App Router 的默认路由匹配规范，避免不必要的重定向。

## 4. 后续建议

- **页面补充**: 目前 `app/` 目录下尚未创建 `about`, `music`, `featured`, `gallery` 等页面文件夹。建议后续根据路由配置创建相应的 `page.tsx` 文件，以免出现 404 错误。
- **移动端优化**: 当前优化主要集中在桌面端，后续可进一步检查移动端菜单 (`isMobileMenuOpen`) 的交互体验。
